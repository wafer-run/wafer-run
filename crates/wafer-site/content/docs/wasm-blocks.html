WASM Blocks

Write blocks in any language that compiles to WebAssembly. WASM blocks declare what services they need via a capability model, and the runtime provides only those. This is the guest-side counterpart to the native Block trait — same interface, but compiled to .wasm and loaded by the runtime at startup.

Writing a Block

Implement the Block interface and register it. The SDK generates the four WASM exports (malloc, info, handle, lifecycle) that the runtime calls.

import {
    Block, BlockInfo, Context, Message, Result, register
} from "wafer-sdk-ts";
import { LoggerClient } from "wafer-sdk-ts/services/logger";

class MyBlock extends Block {
    info(): BlockInfo {
        return {
            name: "@app/my-block",
            version: "1.0.0",
            interface: "processor@v1",
            summary: "Processes incoming messages",
        };
    }

    handle(ctx: Context, msg: Message): Result {
        const logger = new LoggerClient(ctx);
        logger.info("handling: " + msg.kind);
        return msg.continue();
    }
}

register(new MyBlock());
package main

import wafer "github.com/wafer-run/wafer-sdk-go"

type MyBlock struct{}

func (b *MyBlock) Info() wafer.BlockInfo {
    return wafer.BlockInfo{
        Name:      "@app/my-block",
        Version:   "1.0.0",
        Interface: "processor@v1",
        Summary:   "Processes incoming messages",
    }
}

func (b *MyBlock) Handle(ctx *wafer.Context, msg *wafer.Message) *wafer.Result {
    return wafer.ContinueResult()
}

func (b *MyBlock) Lifecycle(ctx *wafer.Context, event wafer.LifecycleEvent) error {
    return nil
}

func main() {
    wafer.Export(&MyBlock{})
}
use wafer_sdk::*;

struct MyBlock;

impl Default for MyBlock {
    fn default() -> Self { Self }
}

impl Block for MyBlock {
    fn info(&self) -> BlockInfo {
        BlockInfo {
            name: "@app/my-block".into(),
            version: "1.0.0".into(),
            interface: "processor@v1".into(),
            summary: "Processes incoming messages".into(),
            instance_mode: InstanceMode::PerNode,
            allowed_modes: vec![],
        }
    }

    fn handle(&self, ctx: &Context, msg: &mut Message) -> Result_ {
        msg.clone().cont()
    }
}

wafer_sdk::export!(MyBlock);

Build to WASM

npx wafer-sdk-ts build -o block.wasm
GOOS=wasip1 GOARCH=wasm go build -o block.wasm .
cargo build --target wasm32-wasip1 --release
# output: target/wasm32-wasip1/release/my_block.wasm

Instance Modes

Control how many WASM instances the runtime creates:

  • Singleton — one instance shared across all flows (default)
  • PerFlow — one instance per flow (isolated state per flow)
  • PerNode — one instance per node in a flow tree
  • PerExecution — a fresh instance for every message (full isolation, higher cost)

WASM ABI

All three SDKs generate the same four WASM exports that the runtime calls:

  • malloc(size) → ptr — allocate memory for the host to write input into
  • info() → i64 — return block metadata as a packed (ptr, len) pointing to JSON
  • handle(ptr, len) → i64 — process a message, return a JSON result
  • lifecycle(ptr, len) → i64 — handle init/start/stop events

The host writes JSON into guest memory via malloc, calls the function, and reads the JSON result back from the returned (ptr, len).

Available Services

WASM blocks can access runtime services through the Context object. Each SDK provides typed clients:

  • Logger — structured logging (debug, info, warn, error)
  • Config — read/write block configuration key-value pairs
  • Database — CRUD operations on collections (get, list, create, update, delete)
  • Network — outbound HTTP requests (GET, POST, etc.)
  • Storage — object storage (put, get, delete, list)
  • Crypto — hashing, signing, token verification

See Clients for usage examples and the Guest SDK for the full API reference.
