Built-in Blocks

wafer ships with ready-to-use infrastructure blocks in the @wafer namespace. All are registered automatically when you call wafer_core::register_all().

@wafer/http-router

Config-driven HTTP router that matches incoming requests by method and path, then dispatches to handler blocks via ctx.call_block(). Returns 404 Not Found when no route matches.

Config:

  • routes — array of route objects, each with:
    • path — path pattern with {var} params and /** wildcard support
    • methods — optional array of HTTP methods (e.g. ["GET", "POST"]); if omitted, matches any method
    • block — the block name to call when matched

Path variables captured by {var} patterns are set as req.param.var in the message meta. Routes are evaluated in order — the first match wins.

{
    "block": "@wafer/http-router",
    "config": {
        "routes": [
            { "path": "/users", "methods": ["GET"], "block": "my-app/list-users" },
            { "path": "/users/{id}", "methods": ["GET"], "block": "my-app/get-user" },
            { "path": "/users", "methods": ["POST"], "block": "my-app/create-user" },
            { "path": "/health", "block": "my-app/health" }
        ]
    }
}

@wafer/security-headers

Adds standard security headers to all responses: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, Referrer-Policy, Strict-Transport-Security, Content-Security-Policy, and Permissions-Policy.

Config: csp — override the default Content-Security-Policy value.

@wafer/cors

Handles CORS preflight (OPTIONS) requests and injects Access-Control-* headers. Supports origin allowlisting, credentials, and configurable max-age.

Config:

  • allowed_origins — comma-separated origins or * (default: *)
  • allowed_methods — allowed HTTP methods (default: GET, POST, PUT, PATCH, DELETE, OPTIONS)
  • allowed_headers — allowed headers (default: Content-Type, Authorization, X-Requested-With)
  • max_age — preflight cache duration in seconds (default: 86400)

@wafer/rate-limit

Per-IP sliding-window rate limiting. Returns 429 Too Many Requests when the limit is exceeded and sets X-RateLimit-Limit, X-RateLimit-Remaining, and Retry-After headers.

Config:

  • max_requests — max requests per window (default: 1000)
  • window_seconds — window duration in seconds (default: 60)

@wafer/readonly-guard

Rejects write operations (create, update, delete actions) with 403 Forbidden when read-only mode is enabled.

Config: readonly — set to "true" to enable (default: "false").

@wafer/monitoring

Tracks request metrics: total count, error count, status code distribution, and path frequency. Exposes a stats endpoint at /_stats or /_monitoring that returns JSON with uptime and request data.

@wafer/auth

Authentication middleware supporting three token types:

  • JWT Bearer tokens — validated via the crypto service
  • API keys (sb_ prefix) — hashed with SHA-256 and checked against the database for revocation/expiration
  • httpOnly cookies — session-based authentication

On success, sets auth.user_id, auth.user_email, and auth.user_roles in the message meta for downstream blocks.

@wafer/iam

Role-based access control. Checks if the authenticated user has the required role. Returns 401 if not authenticated, 403 if the role is missing.

Config: role — required role name (default: "admin").

@wafer/web

Static file server with MIME detection, SPA support, and intelligent caching. Blocks path traversal and dotfile access.

Config:

  • web_root — directory to serve from (default: "./public")
  • web_prefix — URL prefix to strip (default: empty)
  • web_spa — serve index.html for missing routes (default: "false")
  • web_index — index filename (default: "index.html")
  • cache_max_age — cache duration in seconds (default: 3600)
  • immutable_max_age — cache duration for hashed assets (default: 31536000)

Pre-built Chains

wafer-core also registers three chain templates you can reference by ID:

  • http-infra — security-headers → cors → readonly-guard → rate-limit → monitoring
  • auth-pipe — http-infra → auth (requires authentication)
  • admin-pipe — http-infra → auth → iam[role=admin] (requires admin role)

Reference them in your chain config with "chain": "http-infra":

{
    "id": "api",
    "root": {
        "chain": "http-infra",
        "next": [{
            "block": "@wafer/http-router",
            "config": {
                "routes": [
                    { "path": "/api/hello", "methods": ["GET"], "block": "hello" }
                ]
            }
        }]
    }
}
