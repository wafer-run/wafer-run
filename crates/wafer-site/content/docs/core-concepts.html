Core Concepts

Messages

Every request becomes a Message with three fields:

  • kind — the message type (e.g. "action", "event")
  • data — the payload (JSON value)
  • meta — a key-value map of metadata (headers, path params, etc.)

Helper methods on Message:

  • action(kind, data) — create an action message
  • path() — read meta["http.path"]
  • cont(msg) — continue with a modified message
  • respond(msg, status, body) — send a response

Blocks

A Block is any struct that implements the Block trait. Blocks are the fundamental unit of logic — they receive a message, do work, and return a result.

Chains

Chains are trees of blocks. When a message arrives, it flows through the tree. Each node can match on the message (by HTTP method, path, etc.) and route to child blocks. Chains are defined in JSON and loaded at startup.

Actions

Each block returns one of four actions:

  • Continue — pass the message to the next block in the chain
  • Respond — send a response back to the caller immediately
  • Drop — silently discard the message
  • Error — signal an error (the chain's on_error policy decides what happens)
