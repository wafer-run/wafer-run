Flow Configuration

Flows are defined in JSON using the FlowDef format:

{
    "id": "api",
    "summary": "API flow",
    "config": { "on_error": "stop" },
    "root": {
        "flow": "@wafer/infra",
        "next": [{
            "block": "@wafer/router",
            "config": {
                "routes": [
                    { "path": "/api/hello", "methods": ["GET"], "block": "hello" },
                    { "path": "/api/users", "methods": ["POST"], "block": "create-user" },
                    { "path": "/api/items/{id}", "block": "items" },
                    { "path": "/api/files/**", "block": "file-handler" }
                ]
            }
        }]
    }
}

The @wafer/router block is the recommended way to route requests to handler blocks. Routes are evaluated in order — the first match wins. See Built-in Blocks for full config details.

Match Patterns

Flows also support inline match patterns on nodes for lightweight routing without the router block:

{
    "root": {
        "next": [
            { "match": "GET:/api/hello", "block": "hello" },
            { "match": "POST:/api/users", "block": "create-user" }
        ]
    }
}
  • METHOD:/path — match a specific HTTP method and path (e.g. GET:/api/hello)
  • *:/path — match any method
  • {var} — capture a path parameter (available as req.param.var in meta)
  • /** — wildcard suffix, matches any remaining path segments

Flow Config

The top-level config object supports:

  • on_error — error policy: "stop" (abort flow) or "continue" (skip failed block)
  • timeout — request timeout (e.g. "30s", "5m")

Flow Tree Structure

Each node in the flow tree has:

  • block — the block to execute at this node
  • flow — reference to a pre-built flow (e.g. "@wafer/infra", "@wafer/auth-pipe")
  • match — the pattern to match against
  • next — an array of child nodes
  • config — key-value config passed to the block via ctx.config_get()

Node Config

Pass configuration to a block at a specific node:

{
    "block": "@wafer/iam",
    "config": { "role": "editor" }
}

The block reads these values with ctx.config_get("role").
