Deployment

wafer compiles to a single binary with all native blocks, chains, and static assets embedded. WASM blocks and interface definitions are loaded at startup from files or the registry.

Environment Variables

  • PORT — HTTP listen port (default: 8090)
  • RUST_LOG — log level filter (e.g. wafer=info, wafer=debug)

Local

cargo build --release
PORT=8080 ./target/release/my-app

Docker

FROM rust:1-slim AS build
WORKDIR /app
COPY . .
RUN cargo build --release

FROM debian:bookworm-slim
COPY --from=build /app/target/release/my-app /usr/local/bin/
ENV PORT=8080
EXPOSE 8080
CMD ["my-app"]

Fly.io / Railway

Set the PORT environment variable (both platforms inject it automatically) and deploy the binary. No additional configuration needed — wafer reads PORT at startup and binds to 0.0.0.0:$PORT.

Health Checks

Add a health check endpoint to your chain for load balancers and orchestrators:

w.register_block_func("health", |_ctx, msg| {
    json_respond(msg.clone(), 200, &serde_json::json!({"status": "ok"}))
});

Or use the built-in monitoring block which exposes /_stats automatically.

Infrastructure Chains

For production deployments, use the pre-built chains to add security headers, CORS, rate limiting, and monitoring in one line:

{
    "id": "api",
    "root": {
        "chain": "auth-pipe",
        "next": [
            { "match": "*:/api/**", "block": "my-api" }
        ]
    }
}
