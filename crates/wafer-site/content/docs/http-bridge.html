HTTP Bridge

The HTTP bridge converts incoming HTTP requests into wafer messages and routes them to chains.

Method to Action Mapping

  • GET → read
  • POST → create
  • PUT → update
  • DELETE → delete

Meta Keys

The bridge populates these meta fields on each message:

  • http.method — the HTTP method (GET, POST, etc.)
  • http.path — the request path
  • req.query.* — query string parameters
  • req.param.* — path parameters from {var} captures

Response Helpers

// Simple text response
respond(msg, 200, "OK")

// JSON response
json_respond(msg, 200, &json!({"status": "ok"}))

// Error helpers
err_bad_request(msg, "Invalid input")
err_not_found(msg, "User not found")
err_unauthorized(msg, "Token expired")
err_internal(msg, "Something went wrong")

ResponseBuilder

For fine-grained control over the response:

ResponseBuilder::new(200)
    .header("X-Custom", "value")
    .content_type("application/json")
    .body(json!({"ok": true}))
    .build(msg)
