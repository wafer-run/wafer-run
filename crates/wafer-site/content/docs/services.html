Services

Services provide platform capabilities to blocks. Native blocks access services through ctx.services(). WASM blocks use typed clients from the SDK (e.g. DatabaseClient::new(ctx)). See the API Reference for the full method listing.

Database

CRUD operations on collections with filtering, pagination, and raw SQL:

let db = ctx.services().unwrap().database();

// Create
let record = db.create("users", &json!({"name": "Alice", "role": "admin"}))?;

// Read
let user = db.get("users", "user-id-123")?;

// List with filtering and pagination
let opts = ListOptions {
    filter: Some(json!({"role": "admin"})),
    page: 1,
    page_size: 20,
    sort_by: Some("name".into()),
};
let users = db.list("users", &opts)?;

// Aggregation
let count = db.count("users", &json!({"role": "admin"}))?;

// Raw SQL (for complex queries)
let results = db.query_raw("SELECT * FROM users WHERE role = ?", &["admin"])?;

Storage

Binary object storage organized into folders:

let store = ctx.services().unwrap().storage();

// Create a folder
store.create_folder("avatars", true)?;  // true = public

// Upload
store.put("avatars", "alice.png", &bytes, "image/png")?;

// Download
let (data, info) = store.get("avatars", "alice.png")?;

// List objects in a folder
let files = store.list("avatars", &opts)?;

// Manage folders
let folders = store.list_folders()?;

Crypto

Password hashing and JWT token management:

let crypto = ctx.services().unwrap().crypto();

// Password hashing (bcrypt)
let hash = crypto.hash("my-password")?;
crypto.compare_hash("my-password", &hash)?;

// JWT tokens
let token = crypto.sign(&claims, 3600)?;  // 1 hour expiry
let decoded = crypto.verify(&token)?;

// Random bytes
let bytes = crypto.random_bytes(32)?;

Network

Make outbound HTTP requests from blocks:

let net = ctx.services().unwrap().network();

let response = net.do_request(HttpRequest {
    method: "GET".into(),
    url: "https://api.example.com/data".into(),
    headers: vec![("Authorization".into(), "Bearer token".into())],
    body: Vec::new(),
})?;

Logger

Structured logging with optional key-value fields:

let logger = ctx.services().unwrap().logger();

logger.info("Request processed", &[("user_id", "123"), ("duration_ms", "42")])?;
logger.error("Failed to connect", &[("service", "database")])?;

Config

Read and write configuration key-value pairs:

let config = ctx.services().unwrap().config();

let val = config.get("api_key")?;
config.set("last_sync", "2025-01-01T00:00:00Z")?;
