Runtime API

The host runtime that loads blocks, builds flows, and executes messages. Available as a Rust crate (wafer-run), Go module (wafer-run-go), and Node.js package (wafer-run).

Wafer (Runtime)

MethodDescription
Wafer::new()Create a new runtime instance
register_block(name, block)Register a native block instance
register_block_func(name, handler)Register an inline handler function as a block (Rust only)
add_flow(flow)Add a programmatic flow
add_flow_def(def)Add a flow from a JSON FlowDef
resolve()Resolve all flow block references
has_block(type_name) → boolCheck if a block type is registered
registry() → &RegistryAccess the block registry

Block Trait

The core interface every block must implement:

MethodDescription
info() → BlockInfoReturn block metadata (name, version, interface, summary, instance mode)
handle(ctx, msg) → Result_Process a message and return a result
lifecycle(ctx, event) → ResultHandle lifecycle events (Init, Start, Stop)
block_capabilities() → OptionReturn capability restrictions (WASM blocks only)

BlockInfo

FieldTypeDescription
nameStringUnique block identifier
versionStringSemver version string
interfaceStringInterface contract (e.g. "handler@v1")
summaryStringHuman-readable description
instance_modeInstanceModeSingleton, PerFlow, PerNode, or PerExecution
allowed_modesVecAdditional allowed instance modes
admin_uiOptionAdmin UI info (path, icon, title)
runtimeBlockRuntimeNative or Wasm (default: Native)
requiresVec<String>Block names this block may call via call_block(). Empty = unrestricted.

The message object passed through blocks. Available in both runtime and SDK.

Actions

MethodDescription

Context

Passed to handle() and lifecycle(). Provides access to runtime capabilities. Use the clients module for typed access to services.

MethodDescription
call_block(block_name, msg) → Result_Call another block by name. Enforced against the caller's requires list when set; returns PERMISSION_DENIED if the target block is not listed.
is_cancelled() → boolWhether the request has been cancelled (timeout or explicit)
config_get(key) → Option<&str>Read a configuration value from the block's node config

Response Helpers

Convenience functions for building responses. All take a reference to the Message.

FunctionDescription
respond(msg, data: Vec<u8>, content_type)Send a raw response with explicit content type
json_respond(msg, data)Serialize data as JSON and send (sets application/json)
error(msg, code, message)Send an error response with an error code
err_bad_request(msg, message)Invalid argument error
err_unauthorized(msg, message)Unauthenticated error
err_forbidden(msg, message)Permission denied error
err_not_found(msg, message)Not found error
err_conflict(msg, message)Already exists error
err_validation(msg, message)Invalid argument error
err_internal(msg, message)Internal error

Error codes are transport-agnostic. The HTTP listener maps them to status codes (e.g. NOT_FOUND → 404).

ResponseBuilder

MethodDescription
ResponseBuilder::new(msg) → ResponseBuilderStart building a response
.status(code) → SelfSet an explicit status code (e.g. for redirects)
.set_header(key, value) → SelfAdd a response header
.set_cookie(cookie) → SelfSet a Set-Cookie header
.json(data) → Result_Finish with JSON body
.body(data, content_type) → Result_Finish with raw body and content type

A convenience function new_response(msg) is also available as a shorthand for ResponseBuilder::new(msg).

Router (Rust only)

Declarative routing within a block:

MethodDescription
Router::new()Create a new router
retrieve(pattern, handler)Register a GET route
create(pattern, handler)Register a POST route
update(pattern, handler)Register a PUT/PATCH route
delete(pattern, handler)Register a DELETE route
execute(pattern, handler)Register an OPTIONS route
on(action, pattern, handler)Register a route for any action
route(ctx, msg) → Result_Route a message
