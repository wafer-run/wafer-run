Creating a Block

There are two ways to create blocks: native Rust blocks that compile into your binary, or WASM blocks written in any supported language and loaded at runtime.

Native Blocks (Rust)

A native block is a Rust struct that implements the Block trait. No CLI or SDK needed — just write Rust and import it directly.

1. Create Your Block

// blocks/greet.rs
use wafer_run::*;

pub struct GreetBlock;

impl Block for GreetBlock {
    fn info(&self) -> BlockInfo {
        BlockInfo {
            name: "greet".to_string(),
            version: "1.0.0".to_string(),
            interface: "handler@v1".to_string(),
            summary: "Returns a greeting".to_string(),
            instance_mode: InstanceMode::Singleton,
            allowed_modes: Vec::new(),
            admin_ui: None,
        }
    }

    fn handle(&self, _ctx: &dyn Context, msg: &mut Message) -> Result_ {
        json_respond(msg, &serde_json::json!({"greeting": "hi there"}))
    }

    fn lifecycle(&self, _ctx: &dyn Context, _event: LifecycleEvent)
        -> std::result::Result<(), WaferError> { Ok(()) }
}

2. Register It

// main.rs
mod blocks;

let mut w = Wafer::new();
w.register_block("greet", Arc::new(blocks::greet::GreetBlock));

Native blocks compile directly into your binary — no .wasm file, no runtime loading, zero overhead.

For simple blocks you can skip the struct entirely and register an inline handler:

w.register_block_func("greet", |_ctx, msg| {
    json_respond(msg, &serde_json::json!({"greeting": "hi there"}))
});

WASM Blocks

WASM blocks can be written in any supported language and are loaded by the runtime at startup. Use the CLI to scaffold them.

1. Add a Block

wafer block greet --lang typescript
wafer block greet --lang go
wafer block greet --lang rust

This creates a block folder with an entry point and adds it to wafer.json.

2. Edit Your Block

// blocks/greet/index.ts
import { Block, BlockInfo, Context, Message, BlockResult, defineBlock } from "wafer-sdk-ts";

class GreetBlock extends Block {
    info(): BlockInfo {
        return { name: "greet", version: "1.0.0", interface: "handler@v1", summary: "Returns a greeting" };
    }

    handle(ctx: Context, msg: Message): BlockResult {
        return msg.jsonRespond({"greeting": "hi there"});
    }
}

defineBlock(new GreetBlock());
// blocks/greet/index.go
package main

import wafer "github.com/wafer-run/wafer-sdk-go"

type GreetBlock struct{}

func (b *GreetBlock) Info() wafer.BlockInfo {
    return wafer.BlockInfo{
        Name:      "greet",
        Version:   "1.0.0",
        Interface: "handler@v1",
        Summary:   "Returns a greeting",
    }
}

func (b *GreetBlock) Handle(ctx *wafer.Context, msg *wafer.Message) *wafer.Result {
    return wafer.JSONRespond(msg, map[string]string{"greeting": "hi there"})
}

func (b *GreetBlock) Lifecycle(ctx *wafer.Context, event wafer.LifecycleEvent) error {
    return nil
}

func main() {
    wafer.Export(&GreetBlock{})
}
// blocks/greet/index.rs
use wafer_sdk::*;

struct GreetBlock;

impl Default for GreetBlock {
    fn default() -> Self { Self }
}

impl Block for GreetBlock {
    fn info(&self) -> BlockInfo {
        BlockInfo {
            name: "greet".into(),
            version: "1.0.0".into(),
            interface: "handler@v1".into(),
            summary: "Returns a greeting".into(),
            instance_mode: InstanceMode::Singleton,
            allowed_modes: vec![],
        }
    }

    fn handle(&self, _ctx: &Context, msg: &mut Message) -> Result_ {
        json_respond(msg, &serde_json::json!({"greeting": "hi there"}))
    }
}

wafer_sdk::export!(GreetBlock);

3. Build

wafer build

This compiles your block to WebAssembly and outputs it to dist/greet.wasm.

Your wafer.json now includes the new block:

{
  "name": "my-app",
  "version": "0.1.0",
  "blocks": [
    { "name": "hello", "source": "blocks/hello" },
    { "name": "greet", "source": "blocks/greet" }
  ],
  "flows": [
    { "name": "main", "source": "flows/main.json" }
  ]
}

Interface References

A block's input and output fields in block.json accept either an inline JSON Schema object or a string reference to an interface package:

{
  "actions": {
    "POST:/api/data": {
      "input": "github.com/wafer-run/interfaces/http-request@v1.0.0",
      "output": { "type": "object", "properties": { "ok": { "type": "boolean" } } }
    }
  }
}

Use wafer types add to fetch interface packages locally. See the CLI docs for details.
