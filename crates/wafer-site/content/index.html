wafer.run

The wafer thin runtime for tools, apps, and services

Ship secure, sandboxed programs in a single binary. No Docker. No VMs. Just WebAssembly, a tiny runtime, and your code.

For Developers & AI

Create tools that do one thing well

Build small, focused blocks of logic that developers and AI agents can call. Each block takes input, runs your code, and returns a result. Write in Rust, Go, or JavaScript — compile to WebAssembly and it runs anywhere.

Perfect for AI tool-calling: give an LLM a catalog of blocks and let it run the best tool for the job. Each block is sandboxed, deterministic, and designed to be easily testable.

  • Write in Rust, Go, JavaScript, compile to Wasm
  • Structured input/output via JSON Schema
  • Database, storage, crypto, network — all just blocks
  • Publish to the registry, share across teams
my-tool/src/lib.rs 
use wafer_sdk::*;

#[block]
fn handle(msg: Message, ctx: &Context) -> Result {
    // Read input from the caller
    let query = msg.str("query");

    // Use built-in database service
    let results = ctx.db()
        .search("products", &query)?;

    // Return structured data
    ctx.ok(json!({ "results": results }))
}

Secure by Default

WebAssembly sandboxing without the overhead

Every block runs inside a WebAssembly sandbox. It cannot access the filesystem, network, or memory of other blocks unless you explicitly grant it through the service interface. No container escapes. No privilege escalation. No CVE treadmill.

Docker images are hundreds of megabytes, take seconds to start, and need constant patching. A Wasm block is kilobytes, starts in microseconds, and has a tiny attack surface by design. Run untrusted code with confidence.

  • Memory-safe sandbox — no buffer overflows, no escapes
  • Capability-based security — blocks only access what you allow
  • No OS, no kernel, no CVEs to patch
  • Safe to run user-submitted or AI-generated code
Wasm sandbox model 
# What a Wasm block CAN do:
  Compute          pure logic, transforms
  Call blocks    db, storage, crypto, network, etc
  Return data      structured JSON output

# What a Wasm block CANNOT do:
  Read filesystem  no access
  Open sockets     no access
  Spawn processes  no access

# No OS. No kernel. No CVEs to patch.
# Just your code in a memory-safe sandbox.
